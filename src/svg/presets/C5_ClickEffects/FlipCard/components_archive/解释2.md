## mousedown/mouseup的作用

这两个事件控制着整个翻转动画的启动和恢复：[1]

- **mousedown（按下）**：触发卡片缩小 + 翻转到正面的动画
- **mouseup（松开）**：触发卡片恢复 + 翻回背面的动画

```html
<!-- 按下时：宽度和缩放动画 -->
<animate attributeName="width"
         values="200%;200%;100%;100%"
         begin="mousedown" />

<animateTransform type="scale"
                  values="0.5;0.5;1;1"
                  begin="mousedown" />

<!-- 松开时：反向恢复 -->
<animateTransform type="scale"
                  values="1;1;0.5;0.5"
                  begin="mouseup" />
```

## 无限循环的核心：双层热区系统

无限翻转的关键技巧是**双层热区交替机制**。代码中设置了两个热区（热区A和热区B），它们通过mousedown/mouseup事件交替出现在屏幕内外：[1]

### 工作原理

1. **初始状态**：热区B在屏幕内（可点击），热区A在屏幕外（-10000位置）
2. **按下mousedown**：热区A移入屏幕（translate到0），热区B移出
3. **松开mouseup**：热区A移出屏幕，热区B移回
4. **循环**：回到初始状态，可以再次点击

```html
<!-- 热区容器 - 按下后移入，松开后移出 -->
<g>
  <animateTransform type="translate"
                    values="-10000 0;-10000 0;0 0;0 0"
                    begin="mousedown" />
  
  <animateTransform type="translate"
                    values="0 0;0 0;-10000 0;-10000 0"
                    begin="mouseup" />
  
  <!-- 热区A：接收点击触发翻回 -->
  <rect width="750" height="850" />
</g>
```

## 其他关键技术

**顶层拦截热区**：防止动画过程中重复触发，点击后立即移走让下层热区接管。[1]

**touchmove取消机制**：防止用户滑动屏幕时误触发翻转动画。[1]

**透明度时间差**：正面图片在翻转70%时才显示，避免正反面重叠。[1]

这种设计巧妙地利用了mousedown/mouseup事件的时序差异，配合热区的位置切换，实现了看似复杂的无限循环翻转效果。[1]
